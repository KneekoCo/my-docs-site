# How to Build a Simple Task Tracker in Notion

Use this guide to create a Notion-based task tracker that helps you manage your Docusaurus portfolio project.

## Step-by-Step Instructions

### Open Notion and Create a New Page

1. Go to your Notion home page Link: [https://www.notion.so](https://www.notion.so).

2. Click **+ New Page** icon in the left sidebar.

<img src={require('@site/static/img/TaskTracker_NewPage.png').default} width="80%" />
<br></br><br></br>

### Add a Database Table

1. Click **Database** at the bottom of the main screen.

<img src={require('@site/static/img/TaskTracker-AddDatabase.png').default} width="80%" />
<br></br><br></br>

:memo: **Note:** The database pop up will appear, users can select from a few prebuilt tables, build with AI or an empty database. 

2. Select **Empty Database.**

<img src={require('@site/static/img/TaskTracker-NewDatabase.png').default} width="80%" />
<br></br><br></br>

:memo: **Note:** Notion will automatically create a new full-page database table titled “Untitled”.

6. Click the title and give it a name, like `Task Tracker`.

<img src={require('@site/static/img/TaskTracker-NameDatabase.png').default} width="80%" />
<br></br><br></br>

7. You now have a fully functional table to begin adding and organizing your tasks.

### Set Up Your Table Structure

1. Rename the default columns and add new ones as needed:

| Column Name     | Type           | Purpose                                      |
|-----------------|----------------|----------------------------------------------|
|  Task Name     | Title           | Describe the task clearly                   |
|  Status        | Select          | Use: `To Do`, `In Progress`, `Done`         |
|  Priority      | Select          | Use: `High`, `Medium`, `Low`                |
|  Due Date      | Date            | Target completion date                      |
|  Notes         | Text/URL        | Extra info, links, or commands              |

:memo: **Note:** To change a column’s type, click its title → **Property Type** → choose the correct type.

2. Click on the first column,  which is the title column, and reame it `Task Name`.

3. Click **Add Property**

<img src={require('@site/static/img/TaskTracker-AddProperty.png').default} width="80%" />
<br></br><br></br>

4. A menu of different column types with pop up, click the **Select** column type.

:memo: **Note:** For more information about column types see the [Database Column types](#database-column-types) table.

<img src={require('@site/static/img/TaskTracker-AddProperty.png').default} width="80%" />
<br></br><br></br>



5. 

#### Database Column types

| **Column Type**        | **Purpose / Use Case**                                                                 |
|------------------------|----------------------------------------------------------------------------------------|
| **Title**              | Primary identifier for the row (usually the task or item name).                        |
| **Text**               | Freeform text notes or descriptions.                                                   |
| **Number**             | Stores numeric values. Can be formatted (e.g., currency, percentage).                  |
| **Select**             | Choose one option from a predefined list. Ideal for status or category.                |
| **Multi-select**       | Choose multiple tags or categories from a predefined list.                             |
| **Date**               | Assign dates or date ranges (supports time as well).                                   |
| **Person**             | Assign to one or more Notion users.                                                    |
| **Files & media**      | Upload or link files, images, or media.                                                |
| **Checkbox**           | Track binary status like done/undone.                                                  |
| **URL**                | Store clickable links.                                                                 |
| **Email**              | Store email addresses (renders as clickable mailto links).                            |
| **Phone**              | Store phone numbers.                                                                   |
| **Formula**            | Create custom formulas based on other columns (like Excel functions).                  |
| **Relation**           | Link to another Notion database.                                                       |
| **Rollup**             | Summarize or calculate values from related database entries.                           |
| **Created time**       | Auto-generated timestamp when row was created.                                         |
| **Created by**         | Auto-filled with the user who created the row.                                         |
| **Last edited time**   | Timestamp for the last time the row was modified.                                      |
| **Last edited by**     | Shows who last modified the row.                                                       |


### 4. Add Your Tasks
Fill in the table with the tasks for your Docusaurus portfolio. Example:

- Initialize Docusaurus project
- Fix dark/light mode CSS
- Deploy to GitHub Pages
- Add Notion tracker guide (this!)


### 5. Use It!
- Sort by due date or priority
- View progress at a glance
- Toggle to board view with `/board` for a Kanban style if desired


##  Why Use This?
- Organize your portfolio development
- Stay focused and on schedule
- Demonstrate Notion and project management skills to recruiters or clients


##  Pro Tip
Link this Notion tracker in your portfolio or Docusaurus site footer to show off your workflow.
