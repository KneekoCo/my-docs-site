# How to Build a Simple Task Tracker in Notion

Use this guide to create a Notion-based task tracker that helps you manage your Docusaurus portfolio project.

## Step-by-Step Instructions

### 1. Open Notion and Create a New Page

- Go to your Notion home page Link: [https://www.notion.so](https://www.notion.so).
- Click **+ New Page** in the sidebar.

<img src={require('@site/static/img/TaskTracker_NewPage.png').default} width="80%" />

- Name it something like `Task Tracker`.

### 2. Add a Database Table
Open Notion and go to your new page (e.g., Docusaurus Portfolio Tracker).

Click inside the page where it says “Press Enter to continue with an empty page” (if it's blank).

Type/table and you’ll see a dropdown with options.

Select “Table view – Database” from the list.

Notion will automatically create a new full-page database table titled “Untitled”.

Rename the table by clicking the title and typing a new one, such as Portfolio Task Tracker.

You now have a fully functional table to begin adding and organizing your tasks.



### 3. Set Up Your Table Structure
Rename the default columns and add new ones as needed:

| Column Name     | Type           | Purpose                                      |
|-----------------|----------------|----------------------------------------------|
|  Task Name     | Title           | Describe the task clearly                   |
|  Status        | Select          | Use: `To Do`, `In Progress`, `Done`         |
|  Priority      | Select          | Use: `High`, `Medium`, `Low`                |
|  Due Date      | Date            | Target completion date                      |
|  Notes         | Text/URL        | Extra info, links, or commands              |

To change a column’s type, click its title → **Property Type** → choose the correct type.



### 4. Add Your Tasks
Fill in the table with the tasks for your Docusaurus portfolio. Example:

- Initialize Docusaurus project
- Fix dark/light mode CSS
- Deploy to GitHub Pages
- Add Notion tracker guide (this!)


### 5. Use It!
- Sort by due date or priority
- View progress at a glance
- Toggle to board view with `/board` for a Kanban style if desired


##  Why Use This?
- Organize your portfolio development
- Stay focused and on schedule
- Demonstrate Notion and project management skills to recruiters or clients


##  Pro Tip
Link this Notion tracker in your portfolio or Docusaurus site footer to show off your workflow.

# How to Build a Simple Task Tracker in Notion

Use this guide to build a clean and functional task tracker in Notion to manage your Docusaurus portfolio work.

---

## Step-by-Step Instructions

### Step 1: Open Notion and Create a New Page

1. Go to the Notion homepage: [https://www.notion.so](https://www.notion.so)
2. In the left sidebar, click the **+ New Page** icon.

<img src={require('@site/static/img/TaskTracker_NewPage.png').default} alt="New Notion Page" width="80%" />
<br /><br />

---

### Step 2: Add a Database Table

1. From the new page, click **Database** at the bottom of the screen.

<img src={require('@site/static/img/TaskTracker-AddDatabase.png').default} alt="Add Database" width="80%" />
<br /><br />

<Admonition type="note" title="Heads Up!">
The database pop-up allows you to pick a prebuilt table, generate one with AI, or start from scratch.
</Admonition>

2. Choose **Empty Database**.

<img src={require('@site/static/img/TaskTracker-NewDatabase.png').default} alt="Select Empty Database" width="80%" />
<br /><br />

<Admonition type="note" title="Need Column Help?">
See the [**Database Column Types**](#database-column-types) table at the end of this guide for a breakdown of each option.
</Admonition>

3. Click the title and rename the database to something like `Task Tracker`.

<img src={require('@site/static/img/TaskTracker-NameDatabase.png').default} alt="Name Database" width="80%" />
<br /><br />

You now have a blank task database ready for configuration.


### Step 3: Set Up Your Table Structure

Start customizing your table by renaming columns and adding new ones to suit your task tracking needs.

#### Suggested Table Structure

| **Column Name** | **Type**   | **Purpose**                                 |
|-----------------|------------|---------------------------------------------|

| `Due Date`      | Date       | Target completion date                      |

To change a column’s type, click its name → **Property Type** → choose the correct option.
</Admonition>

### Database Column Types

Use this reference to understand each available Notion column type and how to apply it in your task tracker.

| **Column Type**      | **Purpose / Use Case**                                                                 |
|----------------------|----------------------------------------------------------------------------------------|
| **Title**            | Primary identifier for the row (usually the task or item name).                        |
| **Text**             | Freeform text notes or descriptions.                                                   |
| **Number**           | Stores numeric values. Supports formatting (currency, percentage, etc.).               |
| **Select**           | Choose one option from a list (e.g., status).                                           <br /><br /> <img src={require('@site/static/img/TaskTracker-AddOption.png').default} width="40%" /> |
| **Multi-select**     | Choose multiple tags or categories from a list (e.g., `High`, `Low`, `Urgent`).        <br /><br /> <img src={require('@site/static/img/TaskTracker-AddOption.png').default} width="40%" /> |
| **Date**             | Assign single or range-based dates (optionally with time).                             |
| **Person**           | Assign to one or more Notion collaborators.                                            |
| **Files & Media**    | Attach or embed documents, images, videos, or links.                                   |
| **Checkbox**         | Toggle for binary tracking like complete/incomplete.                                   |
| **URL**              | Store clickable web links.                                                             |
| **Email**            | Capture email addresses as mailto links.                                               |
| **Phone**            | Save phone numbers (supports tel: link behavior).                                      |
| **Formula**          | Write custom logic (like calculated deadlines or priorities).                          |
| **Relation**         | Link to another Notion database or item.                                               |
| **Rollup**           | Aggregate or calculate values from related entries.                                    |
| **Created Time**     | Auto-filled timestamp for row creation.                                                |
| **Created By**       | Auto-filled with the user who created the row.                                         |
| **Last Edited Time** | Timestamp for the most recent edit.                                                    |
| **Last Edited By**   | Tracks the user who last edited the row.                                               |
